Architecture Approach — How to build the ASCII animation

The core challenge: rendering names as proportional ASCII art and animating the cycling

A

Canvas-to-ASCII Sampling (Recommended)

Adapts the pretext demo's technique directly. Each name is rendered as large text on a hidden canvas, then sampled for brightness at grid positions. Characters are chosen by brightness AND proportional width (via pretext) to fill each row perfectly.

Pros

  • Proven technique from the reference demo
  • Beautiful proportional ASCII art
  • Any name renders correctly regardless of length
  • Rich visual depth with weight/italic variations

Cons

  • More complex — canvas sampling + pretext palette
  • Palette build takes ~100ms on first load (one-time cost)
B

FIGlet-style Text Art

Use a FIGlet font library (like figlet.js) to render names as blocky ASCII text. Each character maps to a multi-line ASCII block. Simpler but less refined than the pretext approach.

Pros

  • Simpler implementation
  • Classic retro ASCII look

Cons

  • Doesn't use pretext (user's requirement)
  • Limited font choices, blocky appearance
  • Long names may overflow
C

Hybrid: Canvas Sampling + Particle Trails

Same as A, but adds the demo's particle system as a background effect. The name renders via canvas sampling, while particles create ambient movement around and through the text.

Pros

  • Most visually impressive
  • Ambient motion even before pressing Start

Cons

  • Most complex to build
  • Higher CPU usage on mobile
  • May distract from the name reveal

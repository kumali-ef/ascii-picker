Design: PWA & Project Structure

File organization, service worker, and installability

Project Structure

Vite + Vanilla JS
ascii-picker/
├── index.html              # Main page
├── vite.config.js          # Vite config + PWA plugin
├── package.json
├── public/
│   ├── favicon.svg         # App icon
│   ├── icon-192.png
│   └── icon-512.png
└── src/
    ├── main.js             # Entry: init theme, render, attach events
    ├── style.css           # All styles + theme variables
    ├── ascii-renderer.js   # Canvas→ASCII pipeline + pretext palette
    ├── animation.js        # Slot-machine cycling + easing logic
    ├── names.js            # Name CRUD + localStorage
    ├── settings.js         # Duration/theme/grid prefs + localStorage
    ├── modal.js            # Modal open/close/tab logic
    └── theme.js            # System preference detection + toggle

PWA Setup

vite-plugin-pwa
  • Plugin: vite-plugin-pwa — auto-generates service worker + manifest
  • Strategy: generateSW — Workbox auto-caches all assets
  • Manifest: name, icons (192 + 512), theme_color, background_color, display: standalone
  • Offline: Full offline support — all assets cached on first visit
  • Install: "Add to Home Screen" prompt on mobile/desktop

Key Dependencies

@chenglou/pretext — proportional text measurement for ASCII character width matching
vite — build tool + dev server
vite-plugin-pwa — auto-generates manifest + service worker
That's it. No framework, no UI library, no other runtime deps.
